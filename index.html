How this workflow stays safe

  1. Every request is mapped to a runbook before any action is taken.
  2. Plans are generated first so a reviewer can inspect the exact next steps.
  3. Irreversible actions stay blocked until a human approves the job.
  4. Execution evidence is written into the job record for traceability.
  5. Audit entries preserve what happened, when it happened, and why.
